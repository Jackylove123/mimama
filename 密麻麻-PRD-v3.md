# 第一章：产品定义与核心策略

## 1.1 产品定位

密麻麻（MiMaMa）是一款面向中国市场的**极简密码记录微信小程序**。

**一句话定义：** 不需要登录的密码备忘录，打开即用，3步记录，1步查阅。

**核心公式：** 无登录 + 启动暗号 + 腾讯云加密存储 + 导出看广告 = 极简密码工具

**产品Slogan：** 「三步记密码，一键保安全。密麻麻，你的微信密码备忘录。」

**核心理念：** 最好的安全工具，是让你忘记它存在的工具。不登录、不授权、不啰嗦——打开就记，记完就走。你的数据只属于你那串暗号。

## 1.2 v3.0 关键升级（相对v2.0）

| 维度 | v2.0 | v3.0（本版） |
|------|------|-------------|
| 术语 | PIN码 | **启动暗号**（更有仪式感） |
| 修改暗号 | 旧暗号验证→新暗号设置 | **三重确认流程**（验证→刻录→记忆契约） |
| 设置页 | 平铺列表 | **三段式分组**（安全中心/数据资产/辅助与合规）|

| 解锁背景 | 纯白/浅灰 | **动态模糊**（24px高斯模糊，"数据就在门后"） |

## 1.3 用户心智模型

密麻麻不是"云服务"，而是用户手机里的"数字保险箱"。

**三种核心心智：**

1. **完全掌控感** — 用户认为数据属于那串暗号，不属于开发者。不获取头像和授权，极大降低"隐私爬取"的戒心。
2. **即用即走（Utility）** — 用户通常处于"焦虑瞬间"（急需查某个密码或刚注册完新App）。心理预期是：别废话，快给我。
3. **离线错觉** — 虽然数据在腾讯云，但通过暗号加密和无登录设计，给用户营造"数据只在我手机里"的心理安全区。

## 1.4 目标用户

| 类型 | 描述 | 痛点 | 核心诉求 |
|------|------|------|---------|
| 密码焦虑者 | 25-40岁，50+在线账号 | 密码记不住、总在找回密码 | 快速记录、快速查找 |
| 微信重度用户 | 日均3+小时微信 | 不想离开微信生态 | 在微信内解决一切 |
| 不信任云服务者 | 拒绝使用1Password等 | 担心数据上传第三方 | "无登录"的安全感 |
| 轻度用户 | 只有10-30个密码 | 不需要复杂功能 | 简单、够用就好 |

## 1.5 核心价值主张

1. **零门槛** — 不需要登录、不需要授权，打开就能用
2. **极快速** — 3步记录密码，1步找到密码
3. **够安全** — 启动暗号加密 + 腾讯云存储，换手机数据不丢
4. **数据自主** — 随时导出备份，数据永远属于你

## 1.6 差异化策略

| 支柱 | 说明 |
|------|------|
| 无登录 | 市面唯一不需要微信授权的密码工具 |
| 启动暗号 | 类手机解锁体验，0学习成本，有仪式感 |
| 极简路径 | 3步记录 vs 竞品5-7步 |
| 导出美学 | 导出PDF像"存折"一样精美 |
| 微信原生 | 无需下载App，微信内直达 |
| 记忆仪式 | 修改暗号的三重确认流程，强化安全心智 |

# 第三章：启动暗号系统

## 3.1 设计原则

- **术语："启动暗号"** — 不叫"PIN码"、不叫"密码"、不叫"登录"。"暗号"一词自带仪式感和私密感。
- **模仿手机解锁** — 大面积数字键盘，每按一下微震反馈
- **零额外点击** — 输入完6位后自动校验跳转，不需要"确认"按钮
- **暗号即密钥** — 暗号是解密唯一手段，不存储、不上传、无法找回
- **动态模糊背景** — 解锁页背景是模糊后的密码列表，营造"数据就在门后"的沉浸感

### 交互流程

1. 首次打开小程序 → 显示暗号设置页（背景为纯色，因无数据可模糊）
2. 用户输入第一遍暗号 → 圆点逐个填充#176B87
3. 输入完成 → 自动校验 → 圆点清空 → 标题变为「请再次输入以确认」
4. 用户输入第二遍暗号
5. 两次一致 → 自动跳转密码库主页（附带"解锁成功"动效）
6. 两次不一致 → 圆点剧烈左右抖动 + 变红 → 提示「两次输入不一致，请重新设置」→ 清空重来

### 异常流

- **暗号过于简单**：禁止123456、654321、111111、000000、121212等连续/重复序列
- **暗号长度**：固定6位（不提供4位选项，安全底线）
- **忘记暗号**：不提供找回机制。底部永久显示警告文字。

## 3.3 暗号解锁页（日常启动）

### 核心视觉：动态模糊背景

**这是v3最重要的视觉升级之一。**

- **背景**：密码列表页经过24px高斯模糊处理，作为解锁页背景
- **效果**：用户隐约能看到"门后有东西"，对上暗号后"门"打开
- **实现**：
  - 方案A（推荐）：解锁页底层放置密码列表页面，上层覆盖毛玻璃遮罩层
  - 方案B：截图密码列表页后做CSS blur处理作为背景图
- **深色模式下**：暗色模糊 + 暗色毛玻璃面板

### 交互细节

- **触感反馈**：每次点击数字，手机微震（`wx.vibrateShort`）
- **自动跳转**：输入完6位后自动校验，正确→进入主页，错误→抖动+变红
- **正确动效**：圆点变#176B87 + "咔哒"震动 → 毛玻璃面板淡出300ms → 模糊变清晰 → 列表交错滑入（每项50ms）
- **错误动效**：圆点左右抖动300ms + 变红#DC2626 + 错误震动 → 1秒后恢复
- **底部信任声明**：「本地加密 · 不上传云端」永久显示

### 阶梯式锁定策略

| 失败次数 | 后果 | 提示 |
|---------|------|------|
| 1-2次 | 仅抖动 | 无额外提示 |
| 3-4次 | 抖动+红色文字 | 「暗号错误，还剩{N}次尝试」|
| 5次 | 锁定60秒 | 「尝试次数过多，请60秒后再试」+ 倒计时 |
| 5次后 | 显示"忘记了？"入口 | 淡灰色文字，阶梯式引导 |
| 10次 | 锁定15分钟 | 「已锁定，请15分钟后再试」|
| 15次 | 锁定1小时 | 「已锁定1小时，请稍后再试」|

### 暗号遗忘处理

**不提供"找回暗号"入口。** 安全承诺不可妥协。

5次失败后底部显示：「忘记了？您可以选择重置应用，但所有数据将被清除且无法恢复。」

点击 → 二次确认弹窗 → 「继续尝试」/ 「确认重置」(红色) → 清除数据 → 返回暗号设置页

# 第七章：数据安全架构

## 7.1 加密方案

```
用户输入暗号
    ↓
暗号 → Argon2id 派生 → 加密密钥（DEK）
    ↓
每条密码用 AES-256-GCM(DEK) 独立加密
    ↓
密文存储至腾讯云数据库
    ↓
腾讯云仅存储密文，无法解密
```

### 密钥派生
- **算法**：Argon2id
- **输入**：用户暗号（6位数字）
- **参数**：Memory 16-32MB / Iterations 2-3 / Parallelism 1-2
- **输出**：256位DEK
- **存储**：DEK不存储，每次从暗号重新派生

### 数据加密
- **算法**：AES-256-GCM（AEAD认证加密）
- **粒度**：每条记录独立加密（字段级）
- **加密字段**：账号、密码、备注
- **不加密字段**：标题、分类、创建时间（用于搜索/列表索引）
- **IV**：每条记录随机16字节IV

## 7.2 用户身份识别

- **方案**：微信OpenID自动识别（无需用户授权登录）
- **原理**：`wx.cloud.callFunction` 自动获取当前用户OpenID作为数据隔离键
- **数据隔离**：安全规则 `auth.openid == doc._openid`
- **换手机**：同一微信号登录，OpenID不变，数据自动同步 ✅
- **用户体验**：完全无感，不需要任何授权操作

## 7.3 离线缓存策略

| 场景 | 行为 |
|------|------|
| 有网络 | 从腾讯云拉取 → 本地缓存（`wx.setStorage`）|
| 无网络 | 读本地缓存（仅密文） → 提示「显示的是缓存数据」|
| 无网络+未验证暗号 | 无法派生DEK → 提示「需要网络连接」|
| 恢复网络 | 后台自动同步更新缓存 |

**不缓存**：解密后明文密码、DEK密钥

## 7.4 会话安全

| 状态 | 说明 |
|------|------|
| 前台运行 | DEK保持内存，无需重复输入暗号 |
| 切后台 | 5分钟后需重新验证暗号 |
| 被杀死 | 重新打开需验证暗号 |
| 被清理 | 重新打开需验证暗号（数据仍在云端）|
| 12小时未操作 | 需重新验证暗号 |

### CSV表格（P0）
- UTF-8 BOM编码，CRLF换行
- 列：序号/标题/分类/账号/密码/备注/创建时间/修改时间
- 文件名：`MiMaMa_Export_YYYYMMDD_HHmmss.csv`

### HTML密匣（P1）
- 单HTML文件，内嵌CSS+JS，零外部依赖，断网可打开
- 顶部搜索框（本地JS过滤）+ 卡片式列表 + 品牌色#176B87
- 文件≤500KB（1000条以内）

### PDF档案（P2，最美设计）
- 仿真"存折/护照"排版
- 封面：品牌Logo + 「您的离线隐私档案」+ 导出统计
- 内页：左侧蓝色竖线装饰条 + 每条记录 = 序号+分类+图标+标题+账号+密码
- 水印：半透明斜排「密麻麻 · 机密档案」
- 页脚：导出时间 + 「密麻麻：极简，所以极安全。」

## 8.3 激励视频广告策略

### 触发规则

| 条件 | 是否触发广告 |
|------|-------------|
| 24小时内已看过一次 | ❌ 冷却期 |
| 24小时内未看过 | ✅ 必须观看 |
| CSV首次导出 | ❌ 首次免费 |

### 感谢仪式（广告播放完成后）

- 绿色对勾✅动画 1.5s
- 文案：「感谢您的支持！您已成功资助了一次数据加密服务。您的数据依然只属于您。」
- 「正在加密导出...」进度条动画 2s
- 生成完成 → 预览缩略图 → 分享面板

### 心理设计

1. **透明沟通**：引导阶段已告知需看广告
2. **价值交换**：广告换的是"精美加密文档"
3. **24小时冷却**：每天最多一次
4. **零广告页面**：密码库、搜索、详情、添加/编辑、设置、解锁页——**永远无广告**

### 广告规范
- 占屏≤40%高度
- "跳过"按钮始终可见（3秒后）
- 严禁：摇一摇跳转、虚假关闭按钮

### 分享渠道

仅两个：**保存到微信收藏** / **发送给朋友（文件传输助手）**
严禁微博、QQ、邮件等第三方入口。

# 第九章：异常处理与设置页

## 9.1 异常场景

| 场景 | 处理 |
|------|------|
| 忘记暗号 | 5次失败后显示"重置"入口，二次确认后清除数据 |
| 小程序被清理 | 数据在云端不受影响，重开需重新验证暗号 |
| 保存时断网 | 本地暂存，网络恢复后自动上传 |
| 拉取时断网 | 显示本地缓存+提示「网络异常，显示的是缓存数据」|
| 导出时断网 | 弹窗引导开启网络 |
| 腾讯云异常 | Toast「服务暂时不可用，请稍后再试」|
| 暗号验证超时 | 切后台5分钟/12小时未操作 → 回到暗号解锁页 |
| 截屏风险 | 密码默认掩码，8秒自动回掩（不依赖截屏防护） |

## 10.2 安全中心（Security Group）

### 修改启动暗号
- 入口：设置 → 安全中心 → 修改启动暗号
- 流程：三重确认（验证旧暗号 → 刻录新暗号 → 记忆契约长按确认）
- 详见第三章 3.4 节

### 剪贴板自动清除
- 默认30秒，可选20s/30s/60s
- 复制密码后倒计时，到0自动清除
- Toast提示：「已复制，{N}秒后自动清除」

## 10.3 数据资产（Data Group）

### 导出安全档案
- 入口：设置 → 数据资产 → 导出安全档案
- 流程：详见第八章

### 回收站
- 删除操作 → Toast「已移入回收站」+ 3秒内可撤销
- 列表：图标+标题+删除时间+剩余天数
- 操作：恢复 / 彻底删除（红色#DC2626）
- 30天后自动物理抹除
- 空状态：「回收站是空的」

### 存储状态
- 显示：「腾讯云加密存储已就绪」 + 绿色圆点
- 上次同步时间
- **设计目的**：增强用户对"数据不丢失"的信心

## 10.4 辅助与合规（Support Group）

### 暗号记忆测试（P1）

**趣味功能：让用户验证自己是否真的记牢了暗号。**

- 入口：设置 → 辅助与合规 → 暗号记忆测试
- 流程：
  1. 点击进入 → 显示暗号输入界面（同解锁页）
  2. 用户输入暗号
  3. 正确 → 🎉「暗号正确！您记得很牢」+ 小庆祝动画
  4. 错误 → 😰「暗号错误！建议立即复习」+ 提示去修改暗号
- **不消耗失败次数**（这是练习，不是真解锁）
- **不影响锁定状态**
- 建议频率：每周可测试一次（非强制）

### 深色模式（P2）
- 跟随系统 / 亮色 / 暗色
- OLED纯黑优化：背景#000000
- 卡片深色#131A22

### 隐私政策与协议
- 明确声明：「不读取个人信息、不存储原始暗号」
- 包含内容：产品信息、数据收集说明、加密方案说明、广告SDK声明、用户权利、联系方式
