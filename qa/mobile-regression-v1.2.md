# Mimama 真机回归清单（v1.2）

适用范围：暗号改为“可选强提醒”后的版本  
目标：提审前验证新增、导入导出、关闭暗号、重启行为稳定

## 0. 测试环境

1. Android 真机 1 台（建议微信最新版）
2. iOS 真机 1 台（建议微信最新版）
3. 使用“体验版/开发版”安装包
4. 每轮开始前，删除小程序并重新进入（模拟新用户）

## 1. 新用户首启（无暗号）

步骤：
1. 删除小程序后重新打开
2. 观察首屏
3. 进入“我的”

预期：
1. 直接进入首页（不是暗号页）
2. 首页显示未设暗号提醒横条，文案为“你还没设置启动暗号，建议尽快设置以保护本地密码数据”
3. “我的”页显示“设置启动暗号”（不是“重置启动暗号”）

失败判定：
1. 首次进入出现暗号输入页
2. 首页无提醒横条
3. “我的”入口文案错误

## 2. 新增密码 + 引导弹框

步骤：
1. 在无暗号状态下新增一条记录并保存
2. 弹框出现后点击“暂不设置”
3. 再新增一条记录并保存
4. 弹框出现后点击“去设置”

预期：
1. 每次新增成功后都弹“建议设置启动暗号”
2. 关闭弹框后记录已保存
3. 第二次新增后仍会弹（不是只弹一次）
4. 点击“去设置”进入暗号设置页（mode=set）

失败判定：
1. 新增后不弹引导
2. 点“暂不设置”导致保存失败
3. 第二次新增不再弹

## 3. 设置暗号流程

步骤：
1. 在“设置启动暗号”页输入两次一致 6 位数字
2. 返回首页
3. 退出小程序到微信会话列表，再次进入

预期：
1. 提示设置成功
2. 首页未设暗号提醒横条消失
3. 再次进入会先到暗号页，输入正确后进入首页
4. “我的”页入口变为“重置启动暗号”
5. “我的”页出现“关闭启动暗号”入口

失败判定：
1. 设完暗号后仍显示未设暗号横条
2. 重进不需要暗号
3. 入口文案未切换

## 4. 导入导出（无暗号状态）

前置：关闭暗号或新用户未设置暗号

步骤：
1. 在“我的”点导出备份
2. 在“我的”点导入备份

预期：
1. 导入/导出都不弹暗号输入框
2. 导出可正常分享文件
3. 导入成功后统计数字正确（总数/新增/重复/跳过）

失败判定：
1. 无暗号时仍要求输入暗号
2. 导入导出流程被暗号拦截

## 5. 导入导出（已设置暗号状态）

前置：已设置暗号

步骤：
1. 在“我的”点导出备份
2. 输入错误暗号一次
3. 再次输入正确暗号
4. 在“我的”点导入备份并输入正确暗号

预期：
1. 导出/导入前都要求暗号验证
2. 错误暗号给出剩余次数提示
3. 正确暗号后继续执行导入/导出

失败判定：
1. 已设置暗号但不要求验证
2. 错误暗号后仍继续操作

## 6. 关闭暗号

前置：已设置暗号

步骤：
1. 在“我的”点击“关闭启动暗号”
2. 输入当前正确暗号
3. 二次确认“确认关闭”
4. 退出并重进小程序
5. 再测一次导出和导入

预期：
1. 关闭成功提示出现
2. 重进后不再要求暗号
3. 导入导出不再要求暗号
4. “我的”入口变回“设置启动暗号”
5. 首页再次显示未设暗号提醒横条

失败判定：
1. 显示关闭成功但重进仍要暗号
2. 关闭后导入导出仍要暗号

## 7. 重置暗号

前置：已设置暗号

步骤：
1. 在“我的”点击“重置启动暗号”
2. 输入旧暗号
3. 输入两次新暗号
4. 退出重进，分别测试旧暗号和新暗号

预期：
1. 旧暗号不能再解锁
2. 新暗号可正常解锁

失败判定：
1. 旧暗号仍可用
2. 新暗号不可用

## 8. 重启与后台行为

前置：分别在“已设置暗号”和“未设置暗号”两种状态下测试

步骤：
1. 进入小程序后切后台 2 分钟再回来
2. 彻底杀进程后再打开

预期：
1. 已设置暗号：可能需要再次验证（符合安全预期）
2. 未设置暗号：始终可直接进入首页

失败判定：
1. 未设置暗号时被异常拦截到暗号页

## 9. 数据安全回归

步骤：
1. 新增 3 条数据（不同分类、含换行备注）
2. 设置暗号 -> 关闭暗号 -> 再设置暗号
3. 检查首页列表、分类计数、详情内容

预期：
1. 数据不丢失
2. 分类和备注内容不变（含换行）

失败判定：
1. 任意数据缺失或分类错乱

## 10. 记录模板（发现问题时）

请记录以下四项，便于快速修复：
1. 手机型号 + 系统版本 + 微信版本
2. 发生步骤编号（例如 6-2）
3. 实际现象（截图/录屏）
4. 是否可稳定复现（每次/偶发）
